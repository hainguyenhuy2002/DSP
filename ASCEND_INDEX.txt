⚠️ COMMON ISSUES & SOLUTIONS
══════════════════════════════

❌ "npu-smi: command not found"
   ✅ Solution: CANN not installed properly
      Follow Huawei CANN installation guide

❌ "torch_npu not found"
   ✅ Solution: pip install torch-npu

❌ "Out of memory"
   ✅ Solution: 
      - Use smaller model (ChatGLM-6B)
      - Or reduce num_instances
      - Check: num_instances * model_size < 512GB

❌ "NPU utilization very low (<20%)"
   ✅ Solution:
      - Make sure model is on NPU device
      - Check device assignment in code
      - Verify with: npu-smi info

❌ "Model download is slow"
   ✅ Solution:
      - Pre-download on login node
      - Copy to compute nodes
      - Or use cached version

📈 MONITORING DURING PROCESSING
═════════════════════════════════

Real-time monitoring (run in separate terminal):

  watch -n 1 'npu-smi info'

What to look for:

  ✅ Health: OK (for all NPUs)
  ✅ AICore(%): > 80%
  ✅ Power(W): 90-100 per NPU
  ✅ Temp(C): < 60°C (good headroom)
  ✅ Memory-Usage: < 90% (avoid swap)

If something is wrong:

  ❌ Low AICore: Model not using NPU
  ❌ High Temp: Reduce num_instances or power
  ❌ High Memory: Out of VRAM, need optimization
